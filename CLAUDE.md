# 前導實驗系統

## 專案概述
這是一個心理學實驗的前導測試網站，研究「自我揭露」在不同互動對象與評價情境下的差異。

## 實驗設計
2×2 受試者間設計：
- **互動對象**：真人顧問 vs AI 代理人
- **評價情境**：高評價（職涯競爭力評估） vs 低評價（興趣探索）

四個條件：
| 條件 ID | 互動對象 | 評價情境 | 對話者名稱 |
|---------|---------|---------|-----------|
| human-high | 真人 | 高評價 | 戴文琴 顧問 |
| human-low | 真人 | 低評價 | 戴文琴 顧問 |
| ai-high | AI | 高評價 | CareerBot |
| ai-low | AI | 低評價 | MatchBot |

## 前導實驗流程
1. 研究者選擇條件 → 2. 指導語頁 → 3. 對話介面（3 題） → 4. 事後問卷 → 5. 完成頁

### 對話問題（3 題）
1. 請簡單介紹一下你自己。
2. 當你需要做一個重要決定的時候，你通常怎麼進行？
3. 你覺得什麼樣的狀態或成就會讓你覺得自己做得不錯？

### 事後問卷
- 6 題操弄確認（7 點 Likert，1=非常不同意，7=非常同意）
- 1 題開放題：「請問你認為這次對話的目的是什麼？」

## 介面包裝差異
- **真人組**：顧問頭貼（專業真人頭像）、中文姓名+「顧問」稱謂。先自我介紹寒暄，打字時間較長且不均勻（3-7秒）。
- **AI 組**：機器人圖示、CareerBot/MatchBot。簡短說明身份後直接問問題，打字時間平均（1-2秒）。

## 技術架構
- **前端**：React / Next.js (App Router) + Tailwind CSS，部署於 Vercel
- **後端**（尚未實作）：Python FastAPI，負責呼叫 Claude API 生成回應
- **資料庫**（尚未實作）：Firebase Firestore
- 目前對話回應使用 `lib/config.ts` 中的 MOCK_RESPONSES，之後替換為後端 API

- 對話改為 9 題

## 98 對雙極形容詞（7 維度 × 14 對）
定義在 `lib/config.ts` 的 DIMENSIONS 中（正式實驗時使用），維度為：
1. 愚鈍懦弱 ↔ 精明幹練
2. 懶散放縱 ↔ 勤儉恆毅
3. 暴躁倔強 ↔ 溫順隨和
4. 狡詐卑鄙 ↔ 誠信淡泊
5. 內向沉靜 ↔ 外向活躍
6. 計較自私 ↔ 豪邁直爽
7. 悲觀善感 ↔ 樂觀自在

## 開發指令
```bash
npm install
npm run dev     # 開發模式 http://localhost:3000
npm run build   # 正式建置
```

  complete/page.tsx   # 完成頁
lib/
